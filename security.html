Security Policy

Last updated: June 13, 2026

NexusPanel connects to real IT systems, so security is not optional. Please report security issues privately and do not post exploit details in public.

1. Reporting security issues

If you believe you found a security issue in NexusHQ, NexusPanel, the legal site, the public website, an agent, or an integration flow, please report it privately through the contact information on the Legal Contact page.

2. Please do not publicly disclose active issues

Do not publish exploit steps, screenshots, secrets, tenant data, customer data, private URLs, or proof of unauthorized access in public places. Give us a reasonable chance to review and fix the issue first.

3. Responsible testing

Security testing must not:

  • Access, modify, delete, or expose data that does not belong to you.
  • Disrupt service availability.
  • Attack third party services connected to NexusPanel.
  • Use social engineering, phishing, spam, or malware.
  • Attempt to persist access or bypass tenant separation.
  • Run unauthorized commands on devices or agents.

4. Secrets

API keys, agent keys, client secrets, tokens, passwords, database credentials, and private certificates should never be shared publicly. If a secret is exposed, it should be rotated as soon as possible.

5. Account security

Users are responsible for protecting their own accounts and connected services. Use strong passwords, protect your email account, remove users who no longer need access, and keep integration permissions as limited as practical.

6. Tenant separation

Attempting to access another tenant’s data is not allowed. If you accidentally see data that does not belong to you, stop using that path and report it privately.

7. No guarantee

We work to protect NexusPanel, but no system is perfect. Security is a shared responsibility between NexusHQ, users, tenant admins, connected vendors, and device owners.
